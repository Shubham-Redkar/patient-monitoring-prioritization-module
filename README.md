# Patient Monitoring & Sepsis Prioritization Module

A full-stack clinical decision support system that monitors patients in real time, predicts sepsis risk using machine learning, and generates AI-powered clinical explanations to help doctors and nurses act fast.

---

## Features

- **ML-based risk scoring** вЂ” Isolation Forest detects vital sign anomalies; Logistic Regression predicts lab-based sepsis risk
- **Priority triage** вЂ” patients are automatically ranked as Critical, High, Medium, or Normal based on combined model outputs
- **SHAP explainability** вЂ” the top 3 most influential features from each model are extracted and surfaced to clinicians
- **AI clinical summaries** вЂ” Critical/High patients get a 2-sentence clinical explanation generated by Groq (llama-3.1-8b-instant), written in plain clinical language with no ML jargon
- **Alert system** вЂ” alerts fire on Critical always, and on High only when the priority escalates; doctors can acknowledge and clear alerts
- **Manual priority override** вЂ” doctors can override the ML-computed priority with a reason, persisted in the database
- **Role-based access control** вЂ” Admin, Doctor, and Nurse roles with route-level enforcement
- **CSV bulk upload** вЂ” nurses upload hourly patient readings via CSV; every row is Pydantic-validated before insert
- **User management** вЂ” admins can create and delete users with full names and roles

## Getting Started

### Prerequisites

- Python 3.12
- Node.js 18+
- A [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) cluster
- A [Groq API key](https://console.groq.com/)

---

### Backend Setup

**1. Create and activate a virtual environment**

```bash
cd backend
python -m venv env

# Windows
env\Scripts\activate

# macOS / Linux
source env/bin/activate
```

**2. Install dependencies**

```bash
pip install -r requirements.txt
```

**3. Create your `.env` file**

Create `backend/.env` with the following:

```env
# MongoDB
MONGO_URI=mongodb+srv://<user>:<password>@<cluster>.mongodb.net/
MONGO_DB=sepsis_db

# Auth
JWT_SECRET_KEY=your-long-random-secret-here

# Groq
GROQ_API_KEY=gsk_...
GROQ_MODEL=llama-3.1-8b-instant

# Gemini (optional)
GOOGLE_API_KEY=AIzaSy...
GEMINI_MODEL=gemini-2.0-flash
```

To generate a secure `JWT_SECRET_KEY`:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

**4. Run the backend**

```bash
cd app
uvicorn main:app --reload
```

The API will be available at `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

---

### Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

The app will be available at `http://localhost:5173`.

## How the Prediction Pipeline Works

When a prediction is requested for a patient at a given hour:

1. **Full history** is fetched from MongoDB for that patient up to that hour
2. **Vital signs** are passed through the Isolation Forest вЂ” each hour is flagged as anomalous (1) or normal (0). A rolling 4-hour window checks for `sustained_instability` (в‰Ґ 2 anomalies in 4 hours). If fewer than 4 hours of data exist, a rule-based fallback using clinical thresholds is used instead.
3. **Lab results** (if present) are passed through the Logistic Regression model, which outputs a probability and a binary risk label
4. **Priority** is computed from both outputs:

| Lab label | Sustained instability | Priority |
|---|---|---|
| 1 (high risk) | 1 | Critical |
| 1 (high risk) | 0 | High |
| None (no labs) | 1 | High |
| 0 (low risk) | 1 | Medium |
| Any | 0 | Normal |

5. **SHAP** runs on both models to extract the top 3 most influential features for each, formatted as human-readable clinical strings (e.g. `lactate: 3.1 mmol/L (HIGH, normal: <2)`)
6. **Groq LLM** generates a 2-sentence clinical summary for Critical and High patients using those SHAP signals. Responses are cached by signal fingerprint to avoid redundant API calls.

## CSV Upload Format

Nurses upload hourly patient data as CSV. Required columns:

| Column | Description |
|---|---|
| `patient_id` | Integer patient identifier |
| `hour_from_admission` | Hour since admission (0-based) |
| `heart_rate` | bpm |
| `respiratory_rate` | breaths/min |
| `spo2_pct` | % (0вЂ“100) |
| `temperature_c` | В°C |
| `systolic_bp` | mmHg |
| `diastolic_bp` | mmHg |
| `wbc_count` | Г—10вЃ№/L |
| `lactate` | mmol/L |
| `creatinine` | mg/dL |
| `crp_level` | mg/L |
| `hemoglobin` | g/dL |

All rows are validated via Pydantic before any data is written. If any row fails validation, the entire upload is rejected with a detailed error report.
